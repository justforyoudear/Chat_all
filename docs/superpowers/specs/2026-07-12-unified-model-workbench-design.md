# 统一模型工作台设计

## 目标

将现有 ChatALL 界面调整为类似 Diffming 的对比工作台。用户可以在嵌入式面板中登录支持的模型官网，也可以将 API 模型加入同一对话，进行并列比较和分析总结。

## 范围

- 保留现有的内嵌官网登录机制，以 DeepSeek 为第一个提供商。
- 已配置的 API 模型和 OpenAI 兼容的自定义端点可加入当前对比。
- 以精简的添加模型入口和当前对话中已加入的模型，替代旧版全量 Bot 底部图标栏。
- 提示词优化使用可用的 API 模型；没有可用模型时显示可操作的应用内提示，不再只在控制台抛错。
- 保留已有对话和设置数据。

## 非目标

- 本次不删除旧 Bot 实现，也不迁移已持久化的 Bot 记录。
- 本次不新增 DeepSeek 以外的官网登录集成。
- 本次不改动构建、发布配置，也不新增依赖。

## 模型流程

1. 用户将官网提供商或已配置的 API 模型加入当前对话。
2. 每个模型占用一个对比面板：官网提供商渲染嵌入式 webview，API 模型渲染本地流式响应。
3. 一个提示词会分发给所有已选择的 API 模型；官网面板沿用其现有网页交互。
4. 响应保留给已有的比较、共识分析和总结机制使用。

## 提示词优化

提示词优化优先使用当前对话中已选定的 API 模型；若不存在，则使用其他已配置 API 模型。若仍无可用模型，界面提示用户在设置中添加 API 模型，且不得产生未处理的控制台异常。

## 界面

底部只包含输入框、发送按钮、提示词工具和紧凑的添加模型入口，不再渲染历史提供商的图标列表。空对话模型面板需区分官网模型与 API 模型，但不显示未使用的旧提供商控制项。

## 错误处理

- 未配置 API：显示本地化、可操作的配置说明。
- 官网 webview 初始化失败：保留该面板并显示其现有错误状态，不能影响 API 面板。
- 单个模型响应失败：仅在该模型面板中隔离失败状态。

## 验收

- 所有变更文件通过针对性 lint 和 Prettier 检查。
- Electron 开发模式可启动应用。
- 手工验证：DeepSeek 官网登录可在面板中打开；自定义 OpenAI 兼容 API 模型可加入同一对话；旧底部模型图标栏不存在。
- 有 API 配置时提示词优化成功；没有可用 API 时显示明确的配置指引。
