# ChatALL 多模型官网对话改造总结

## 目标

将项目从旧的模型图标栏改造为固定多面板对比工作台：保留官网登录后的内嵌对话，同时支持 OpenAI 兼容 API 模型，并可由底部统一输入框同时发送。

## 已完成

1. 官网模型接入：DeepSeek、Kimi、通义千问、智谱清言、豆包作为官网 WebView 模型注册。
2. 固定面板：对话区始终展示官网面板与 API 面板；选择官网模型后先显示登录浮层，关闭后在对应卡片中复用登录会话。
3. 底部发送：移除旧图标栏，当前收藏并启用的模型是统一发送目标。
4. 二次发送：官网模式不再轮询并等待网页回复，避免每个模型的锁被网页抓取任务占用两分钟。
5. Kimi 专用发送：聚焦 `.chat-input-editor` 后通过 Electron 原生字符事件输入，等待官网解除 `.send-button-container` 的禁用状态再点击。
6. WebView 生命周期：注册前的短暂空窗会重试；卡片及登录浮层均拦截新窗口并在当前 WebView 内导航。
7. 错误稳定性：移除损坏的 `vue3-shortkey` 用法，改为项目内部安全指令；开发环境过滤非致命的 `ResizeObserver loop` 遮罩。

## 验收标准

- 两个已登录官网面板都能收到同一条底部消息。
- 同一会话连续发送两条消息时，第二条不等待第一条网页回复。
- 官网跳转不创建额外 BrowserWindow。
- 未配置 API 密钥时，仅提示 API 功能不可用，不阻断官网模型发送。

## 状态
